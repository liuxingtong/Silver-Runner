## 一、CLD 回路简介

### 1.1 研究背景

徐家汇适老化改造采用**人机共生城市系统**建模，通过 CLD（因果回路图）描述老年人聚集、街道舒适、认知恢复、代际互动等变量之间的因果关系，并识别人机装置部署的优先路段。

### 1.2 核心回路类型

| 类型 | 含义 | 示例 |
|------|------|------|
| **R 强化回路** | 正反馈，变量相互增强 | R1 社交活力、R4 认知恢复 |
| **R 注入链** | 外生变量注入，驱动下游 | R2 步行可达、R3 环境舒适 |
| **R 恶性回路** | 负向正反馈，系统恶化 | R_bad 交通诱导（车多→噪声→不愿步行→更多人开车） |
| **B 平衡回路** | 负反馈，系统自调节 | B2 拥挤调节、B3 认知超载 |
| **B 抑制回路** | 外生抑制，削弱目标 | B5 文化摩擦、B6 隔代锁定 |

### 1.3 关键枢纽节点

- **N02**：老年人聚集强度，多数回路的交汇点
- **N18**：人机共生干预指数，设计变量（仿真时拨动 0/0.6）
- **N09**：街道舒适度，MVR 版为内生变量，闭合 B2 拥挤调节回路
- **N10**：车流量代理，MVR 版为内生变量，闭合 R_bad 交通诱导回路

### 1.4 人机干预机制

人机干预**不改变 CLD 拓扑**，只改变边的传递函数参数。干预开启时：

| 边 | 修改内容 |
|----|----------|
| N_YP→N_IG | 虚线(权重≈0) → 实线(激活) |
| N_GC→N16 | 负向抑制系数减弱 |
| N14→N15 | 倒U型峰值右移（系统能承受更高认知复杂度） |
| N03→N14 | 正向传递增益提高 |

6–9 个月出现相变点，N_IG 达临界密度后自我强化，R4 与行为回路产生协同效应。

## 二、平台数据与衍生指标清单

### 2.1 数据源总览

| 数据源 | 存储路径 | 空间形态 | 坐标系 |
|--------|----------|----------|--------|
| 街景分割 | `streetview_images/*.csv` | 点 (lon, lat) | WGS84 |
| 人口栅格 | `population/*.tif` | 100m 栅格 | WGS84 |
| POI | `poi_data/*.csv` | 点 (lon, lat) | GCJ→WGS84 |
| 房价小区 | `house_data/*.xlsx` | 点 (lon, lat) | WGS84 |
| 用地 | `landuse_data/*.geojson` | 多边形 | WGS84 |
| 路网 | `road/*.graphml` | 边 + 节点 | WGS84 |

### 2.2 街景衍生指标

| 指标 | 含义 | 计算方式 |
|------|------|----------|
| GVI | 绿视率 | vegetation + terrain |
| SVF | 天空可见度 | sky |
| Shade_Comfort | 遮蔽舒适度 | (vegetation + building + bridge + tunnel - sky) 归一化 |
| Motor_Pressure | 机动车压力 | car + bus + truck + motorcycle + road |
| Shannon_H | Shannon 多样性 | -Σ(pᵢ·ln(pᵢ))，按 7 大语义类别 |
| Complexity | 视觉复杂度 | 非零类别数 × Shannon_H，归一化 |
| Ped_Friendly | ART 兼容 | sidewalk + bench + pedestrian area + street light - barrier - fence |
| ART_Score | ART 综合恢复潜力 | 0.3×Fascination + 0.25×Being_Away + 0.2×Coherence + 0.25×Ped_Friendly |

### 2.3 人口、POI、用地、路网

- **人口**：`pop_65plus`、`pop_15_59`、`pop_0_14` 栅格采样
- **POI**：边中点 100m 半径内 `poi_social`、`poi_medical`、`poi_culture`、`poi_total`
- **用地**：网格 `shannon`、`green_rate`、`n_types`
- **路网**：`highway_type`、`length`、`maxspeed`、`lanes`，`edge_traffic_pressure` = motor_flag × (lanes/4) × (maxspeed/60) 归一化

### 2.4 CLD 变量与数据映射

**说明**：路网边计算（`analysis/road/cld.py`）使用 N08 表示街道舒适度；SD 仿真与 CLD 模型统一使用 N09。两者为同一概念，公式均为 GVI + Shade_Comfort + Ped_Friendly（0.4:0.3:0.3）。

| CLD 节点 | 含义 | 平台数据/指标 | 所属回路 |
|----------|------|---------------|----------|
| N01 | 社交场所密度 | poi_social + poi_culture + 0.8×poi_medical | R1 |
| N02 | 老年人聚集强度 | pop_65plus 归一化 | R1, R5, B2 |

| N10 | 车流量代理 | edge_traffic_pressure | R_bad |
| N11 | 噪声与污染代理 | 同 N10，叠加 Motor_Pressure | R_bad |

## 四、SD 仿真边函数清单（MVR 版）

### 4.1 变量角色

| 角色 | 变量 | 说明 |
|------|------|------|
| **内生** | N02, N09, N10, N15, N16, N_IG, N19, N20 | 8 个，由方程更新 |
| **外生** | N01, N03, N05, N07, N14, N17_base, N_YP, N_GC, N10_base | 固定输入 |

### 4.2 控制参数

| 参数 | 默认值 | 含义 |
|------|--------|------|
| N18 | 0.0 | 人机干预强度（0=无，0.6=干预） |
| alpha | 2.0 | N18 对 N_YP→N_IG 的放大系数 |
| beta | 0.15 | N_IG→N15 注入强度 |
| gamma | 0.6 | N18 对 N10 的削弱系数 |
| delta | 0.5 | N18 对 N17 的削弱系数 |

### 4.3 一步更新方程（离散化）

**N10（车流量）**：
```
N10_eq = clip(N10_base + k10_NYP·N_YP, 0, 1) × (1 - gamma·N18)
dN10 = 0.30 × (N10_eq - N10)
```

**N09（街道舒适度）**：
```
N09_eq = clip(N09_base + k09_N07·N07 - k09_N02·N02 - k09_NYP·N_YP - 0.10·N10, 0, 1)
dN09 = 0.20 × (N09_eq - N09)
```

**N_IG（代际互动）**：
```
N_IG_target = clip(N_YP × (1 + alpha·N18), 0, 1)
dN_IG = tau_IG × (N_IG_target - N_IG)
```

**N17 有效值**：
```
N17_eff = clip(N17_base × (1 - delta·N18) + 0.15·N10, 0, 1)
```

**N15（认知储备）**：
```
dN15 = k14·N14 - k17·N17_eff·N15 + beta·N_IG - k15_decay·N15
```

**N16（探索意愿）**：
```
dN16 = k16_N15·N15 - k16_GC·N_GC - k16_decay·N16
```

**N02（聚集强度）**：
```
N02_eq = clip(k02_social·N01·N03 + k02_access·N05 + k02_comfort·N09, 0, 1)
dN02 = 0.15×(N02_eq - N02) - k02_crowd·N02²
```

**N19, N20（R5 人机共生，仅 N18>0）**：
```
dN19 = k19_N02·N02·N18 - k_b4·N18·N19 - k19_decay·N19
dN20 = k20_N19·N19 - k20_decay·N20
```

## 五、改造优先级计算（CSVI 模型）

采用 **CSVI（认知-社会脆弱性指数）** 计算边级改造优先级，CSVI 越高越优先改造。

### 5.1 公式

```
CSVI_i = (E_i × S_i) / (AC_i + 1)
```

- **E**（暴露度）：高认知风险老年人在此路段驻留的加权密度
- **S**（敏感性）：环境认知伤害 + 代际接触浪费
- **AC**（适应能力）：物理缓冲 × 社会缓冲

分母接近零时 CSVI 趋向极大，表示该路段几乎无设计缓冲，优先级最高。

### 5.2 暴露度 E

五类老年人画像的加权在场密度，权重反映认知风险：

| 画像 | 权重 | 在场密度 nₖ |
|------|------|-------------|
| 独居高龄老人 | 1.0 | pop_65plus × (1 + poi_social_norm + poi_market_norm) |
| 医疗重度使用者 | 0.9 | pop_65plus × (1 + poi_medical_norm) |
| 隔代照料者 | 0.6 | pop_65plus × (1 + poi_sport_norm) |
| 晨练社群组织者 | 0.5 | pop_65plus |
| 活跃新老年（60–64岁） | 0.4 | pop_60_64 |

`E_i = normalize(Σ wₖ × nₖᵢ)`，结果归一化至 [0,1]。

### 5.3 敏感性 S

两种伤害叠加：`S_i = S_env_i + S_contact_i`

- **S_env**（环境认知伤害）：`Motor_Pressure × (1 - Complexity)`，车流噪声激活生理应激，视觉复杂度可部分对冲  
  - 数据来源：街景分割 → `Motor_Pressure`（car+bus+truck+motorcycle+road）、`Complexity`（非零类别数×Shannon_H）
- **S_contact**（代际接触浪费）：`pop_15_59_norm × (1 - Q)`，其中 `Q = Ped_Safety × Ped_Friendly`  
  - 数据来源：街景 `Ped_Safety`、`Ped_Friendly`；人口 `pop_15_59`

### 5.4 适应能力 AC

`AC_i = AC_phys_i × AC_social_i`

- **AC_phys**：`Ped_Infra × N09 × alpha`，步行设施 × 街道舒适度 × 路网 α 连通指数  
  - 数据来源：街景 `Ped_Infra`（sidewalk+crosswalk+curb cut+pedestrian area+lane marking）、N09（GVI+Shade+Ped_Friendly）
- **AC_social**：`Ped_Friendly × (poi_social + poi_culture) / max(...)`  
  - 数据来源：街景 `Ped_Friendly`；POI 分类 `poi_social`、`poi_culture`

**AC 归一化与中心化**：原始 AC_phys、AC_social 先 min-max 归一化至 [0,1]，再对 AC_combined = AC_phys_norm × AC_social_norm 做二次归一化并中心化（减均值），用于 CSVI 分母。

**说明**：若 `merged.csv` 为原始分割数据（无 Motor_Pressure、Ped_Infra 等列），流水线会自动调用 `streetview_osm_visualization.compute_all_indicators` 计算衍生指标后再挂载。

分母保护：`CSVI = E × S ÷ (AC+1)`，AC+1 避免除零且更稳健。

### 5.5 计算流程与进度

流水线步骤（含 CSVI 子步骤）：

1. 加载路网  
2. 挂载街景指标（含 Ped_Infra、Ped_Safety）  
3. 挂载 POI 分类（含 poi_sport、poi_market）  
4. 挂载用地网格  
5. 挂载人口栅格（含 pop_60_64）  
6. 计算车流量代理  
7. 计算 CLD 变量  
8. 计算暴露度 E  
9. 计算敏感性 S  
10. 计算适应能力 AC  
11. 计算 CSVI 综合得分  
12. 写入边属性  
13. 导出结果  

输出同时包含 `csvi_E`、`csvi_S_env`、`csvi_S_contact`、`csvi_AC_phys`、`csvi_AC_social`，便于诊断每条路段得分高的原因。

### 5.6 预计算脚本

运行 `python scripts/precompute_cld_priority.py` 可提前执行完整流水线（含 E、S、AC、CSVI），结果保存至 `cache/cld_priority.pkl`。仪表盘启动时若存在该缓存，将直接加载，无需在界面内重新计算。

建议在首次使用或数据更新后运行一次，以加速仪表盘加载。

| 类型 | 条件摘要 |
|------|----------|
| **激活** | `priority`（CSVI）Jenks **最高档**；且 **W_elder 或 W_work** 至少其一为 Jenks **最高档**。子类：仅老年轴高 / 仅工作轴高 / 双高 |
| **临界** | CSVI **中档**；**S_contact / (S_env+S_contact) > 0.6**；W_work **高档**；W_elder **非高档** |
| **瓶颈** | 非激活、非临界；**csvi_AC_phys ≤ 全域下中位数**（较低约 **50%**，与「后 10%」旧口径不同）；且该边落在 **「两簇激活衔接带」**上（见下） |

**地图绘制（与分类独立）**：三类节点的 **SVG 标记**均为上述分类结果的**全量**绘制（不再按类内 priority 取前 10%）。底图路网仍为全表中 **priority 最高约 1%** 的边段合并绘制（`MAP_DISPLAY_TOP_FRAC`，性能考虑），与节点标记覆盖范围无关。

#### 瓶颈节点 · 「衔接带」算法（替代几何最短路）

不再使用「边权 = 线段欧氏长度」的 **Dijkstra 最短路** 及路径回溯。

1. 取所有**激活边**的端点顶点（去重）。在 **平面近似坐标**（经度 × cos(纬度)、纬度）上对顶点做 **k-means（k=2，Lloyd）**，得到与空间分布一致的两簇，分别标为簇 0 / 1，作为 BFS 种子。若 k-means 退化（全体重合、单簇等），则**回退**为：按 **经度** 排序后 **前后两半** 各标一簇（与旧版兼容）。  
2. 在**全路网**上做 **无权多源 BFS**：顶点继承**先到**的簇标签（图论跳数意义下的 Voronoi；**仍不**用地理边长，路网疏密会使分界在地图上略有形变）。  
3. **衔接边集** = 全图中 **两端均已染色且簇号不同** 的边（两簇势力分界带）。  
4. **瓶颈** = 衔接边 ∩ { AC_phys ≤ 全域下中位 }。  

若激活端点少于 2 个，或种子未能形成两簇，则不产生该类衔接边（瓶颈可能为空）。
